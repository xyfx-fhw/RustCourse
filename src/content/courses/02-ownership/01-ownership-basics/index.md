# 速览

所有权是 Rust 最独特的特性，它使得 Rust 无需垃圾回收器就能保证内存安全。理解所有权系统是掌握 Rust 的关键。

本节课程分为三个部分：

- **所有权规则** - 了解所有权的三条核心规则
- **移动语义** - 理解值的移动和所有权转移
- **克隆数据** - 深拷贝数据的方法

# 什么是所有权？

所有权是 Rust 管理内存的方式。在传统的编程语言中，内存管理通常有两种方式：

1. **手动管理**（如 C/C++）- 程序员负责 `malloc/free`
2. **垃圾回收**（如 Java、Python）- 运行时自动清理

Rust 选择了第三条路：**所有权系统**。编译器在编译时检查一系列规则，如果违反规则，程序将无法编译。

# 所有权的优势

✅ **内存安全** - 无需垃圾回收器
✅ **零成本抽象** - 运行时无额外开销
✅ **无数据竞争** - 编译时防止并发问题
✅ **可预测性能** - 内存释放时机确定
